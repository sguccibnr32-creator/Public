### §4.2 problem statement

PowerShell 5.1 における .Count property の semantics 観察:

```
single-object case  : 
  $obj = "single string"
  $obj.Count            # returns: empty / undefined / null (実装依存)
  
collection case     :
  $arr = @("a", "b", "c")
  $arr.Count            # returns: 3 (integer)
  
single-object as collection (array coercion) :
  $obj = "single string"
  @($obj).Count         # returns: 1 (integer, forced array coercion)
  
empty collection case :
  $empty = @()
  $empty.Count          # returns: 0 (integer)
  @($empty).Count       # returns: 0 (integer, forced array coercion)
```
